    .NET으로 웹 애플리케이션 개발하기

    July 6, 2025 · 6 min · Hyun, Junyong

    개요.

    서강대에 입학하고 전공이 생각보다 적성에 맞지 않아서 2전공으로 인공지능을 열심히 공부했다.

    그러던 중 25-1학기(막학기)에 ‘AI융합캡스톤디자인과 창업’이라는 필수전공과목을 수강했다.

    서로 다른 과에서 온 학생들이 협력해서 AI를 활용한 사업 아이템을 개발하고 시연해야 하는 수업이었는데, 내가 조장을 맡게 되었다.

    내가 속한 조의 인원수는 4명이었는데, 그 중 둘은 프로그래밍 기술이 거의 없었고, 나머지 한 명은 파이썬과 웹 디자인 실력이 좋았지만 시간이 많지 않은 친구였다.

    나는 조원들에 비해서 과제에 투자할 수 있는 시간이 더 많았기 때문에 정말 다행이었다.

    우리 조의 주제는 여러 번 바뀌었는데, 처음에는 민원인이 전화를 걸면 바로 담당부서로 연결되게 만들어주는 일종의 ‘AI 교환원’을 개발하려고 했다.

    브레인스토밍 과정에서 음성 인식과 감정 분석 기능, 임베딩을 통한 부서 연결 등을 생각했었는데, 시연을 위한 실제 전화번호를 개통하는 것이 불가능했고, 교수님에게 주제가 조금 약하다는 평가를 받았다.

    그래서 전화 연결 기능은 살리되, 평소에 내가 정말로 ‘있었으면 좋겠다’하고 생각했던 것들이 무엇이었는지 되짚어 보았다.

    그러다가 학교 측 행정 처리에 용무가 있을 때마다 어떤 번호로 전화를 해야 하는지 몰라 인터넷을 뒤져야 했던 경험이 떠올랐다.

    사실 복학한 직후에 서강대에 관련된 용건을 입력하면 자동으로 담당자 전화번호가 뜨게 하는 시스템을 개발하려고 생각했던 적이 있었는데, 그때는 혼자 담당자-담당업무-전화번호 데이터를 모두 정리하는 게 불가능했다.

    그러나 서강대 캡스톤경진대회 준비를 위해 필요하다고 주장하면 학교 측에 정식으로 관련 데이터를 요청할 수 있겠다는 생각을 했고, 조원들도 서강대 학생들을 위한 앱을 만들어보자는 내 의견에 긍적적으로 반응해줬다.

    추가 브레인스토밍 과정에서는 데이터베이스와 연동을 통한 ‘실시간 hot 키워드’ 기능과 학생들의 여론을 수치화할 수 있는 ‘학생 청원’ 기능, ‘서강대 건물 내부 혼잡도 기능’ 등의 아이디어가 나왔다.

    실제 개발에 사용할 수 있었던 시간은 약 3주 반 정도였기 때문에 최선을 다해 노력했지만 경진대회에서는 ‘AI 알로스 - 서강대 학생들을 위한 도우미’ 와 ‘학생 청원’, ‘실시간 hot 키워드’ 3개 기능만 구현이 된 채로 발표했다.

    (가장 왼쪽에 흰옷을 입고 있는 사람이 접니다)

    웹 애플리케이션을 개발하는 방법은 정말 다양하다.

    웹 호스팅 방법만 해도 하나가 아니고, 웹 개발에 사용할 수 있는 프레임워크도 굉장히 다양하다.

    최종적으로는 앱 형태로 만드는 것이 목표였는데, 앱의 종류도 다양하다.

    불행인지 다행인지 웹 디자인을 제외하면 개발부터 디버깅, 인덱싱, 최종 점검까지 시작부터 끝까지 내가 모든 작업을 주도해서 마무리했기 때문에 내게 익숙한 방식으로 진행할 수 있었다.

    웹 애플리케이션을 완성한 후 웹 앱 형태로 만드는 단계에서는, 내가 Android만 알고 ios에 대해서는 잘 모르기 때문에 안드로이드용 앱만 만들 수 있었다.

    다만 http://junhsogang.godohosting.com 으로 직접 웹에서 접속하면 아이폰으로도 사용 가능하다.

    완성된 앱의 이름은 ‘포켓 서강’이다. 서강대 생활에 필요한 모든 것을 주머니에 넣고 다니는 것처럼 편하게 제공한다는 의미를 담아 지었다.

    목표.

    첫 회의에서 25-1학기부터 서강대가 ‘서강챗봇’ 프로토타입을 GPT 기반으로 만들었다는 것을 알게 되었기 때문에 ‘서강대 공식 챗봇보다 더 유용한 앱 만들기’가 목표가 되었다.

    경진대회 시점에서는 어느 정도 증명한 것 같다.

    개인적으로 목표로 했던 기능들은:

    1.   LLM을 적용한 AI 도우미 - 원터치 담당자 연결 기능, 원터치 네이버 지도 연결 기능을 포함

    2. 공감/비공감 기능이 있는 학생 청원

    3. 서강대 세인트 아이디/비번과 연계한 회원가입/로그인

    4. DB에 저장된 질문을 임베딩한 후 벡터 DB와 비교해 기간별 인기 키워드 구현 자동화

    5. 건물 와이파이 라우터에 접속해 있는 실시간 이용자 수로 건물별 혼잡도 추정

    6. 서강 뉴스와 연동

    등이 있었는데 아쉽게도 목표한 것을 모두 이루지는 못했다.

    개발 과정에서 사용한 기술 스택.

    개발 환경(IDE): Visual Studio 2017

    개발 플랫폼: .NET Framework 4.6.1

    웹 애플리케이션 개발 프레임워크: ASP.NET

    안드로이드 앱 개발 환경: Android Studio

    사용 언어: C#, Kotlin, html, CSS, TSQL

    DB SSMS: 마이크로소프트 SSMS 18

    AI 알로스에게 제공한 데이터:

    1.   2025년 학사일정

    2. 서강대 모든 건물의 네이버지도링크 데이터

    3. 동아리 데이터 + 대표자 전화번호 + 인스타 아이디

    4. 교내 후생시설 위치

    5. 서강대 학식메뉴 데이터

    6. 서강대 전화번호부-번호별 담당업무 데이터 (약 890개)

    7. 2024년 2학기 모든 수업 강의평가 데이터

    8. 서강대 교환학생 관련 공지 데이터

    9. 서강대 교육이념

    10. 서강대 전공 관련 데이터

    구현 과정.  

    사실 5월 초까지도 진짜 웹사이트하고 앱을 만들 필요는 없는 줄 알았는데 교수님이 면담 시간에 당연히 해야 한다는 식으로 말씀을 하셨기 때문에 그때부터 급하게 3주 동안 열심히 개발을 했다.

    (급하게 진행했던 첫 회의. 총원 4. 1명 연구실 인턴 활동으로 불참. 1명 당일 병으로 불참 통보. 참여인원 2. ㅋㅋㅋ 그래도 프로그래밍 기술이 있는 1명은 참석해서 굉장히 유익한 회의 시간이었다.)

    최종 결과물인 ‘포켓 서강’이 나오기까지의 구현 과정을 정리해보면:

    1. 테스트용 웹서버 구동
    2. Visual Studio .NET Framework 4.6.1 기반 웹 폼으로 만들기 시작
    3. webconfig에 ConnectionString 작성 (DB 연동에 필요함)
    4. 데이터베이스 스키마 설계 후 테스트용 웹서버와 연동
    5. 회원가입/로그인 페이지 완성
    6. Gemini LLM을 사용한 실험용 챗봇 페이지 완성
    7. 실험 후 고정 ip를 위해 윈도우호스팅+웹서버 신청
    8. 초기 welcome 페이지, 학생청원 페이지, 월간 인기 키워드 구현
    9. DB와 연동한 AI 알로스 시스템 완성
    10. FTP 연결을 통해 완성된 파일 웹에 업로드
    11. 웹 디자인 작업(조원이 했음)
    12. 페이징 작업, 디자인 최종 수정
    13. Android Studio로 안드로이드용 웹 앱 만들고 .apk로 release
    14. 조원들과 성능 테스트 후 개선사항 정리
    15. 개선사항(학생청원 삭제기능, 제목 길어짐 방지, 강의실 축약어로 입력시 구분 못함 등) 해결
    16. 캡스톤경진대회 발표용 웹 애플리케이션 완성

    정도로 요약할 수 있다.

    AI 알로스 구현에 필요한 데이터 정리는 조원들이 했다.

    기간별 인기 키워드 구현의 경우 자동화가 목표였기 때문에 DB에 저장된 사용자의 질문을 임베딩하고 미리 정리해 놓은 예상 질문항목 벡터 DB와 비교해서 top 10만 table에 저장하는 부분까지 완성하려고 했지만 시간이 모자라서 발표날까지 끝내진 못했다.

    (완성해서 올려놓은 aspx, aspx.cs 파일들)

    SSMS의 경우에는 mssql 웹 서버를 사용했고, 사용자가 AI알로스에게 한 모든 질문은 테이블에 저장되는 시스템이라서 만약 상용화된다면 학과별, 학년별로 무엇을 궁금해하는지 데이터 분석을 통해 가치 있는 시각 자료를 추출할 수 있다.

    서강대 건물 내 혼잡도 기능은 관련 데이터를 학교 측으로부터 받아 올 수가 없었기 때문에 구현하지 못했다.

    .NET의 경우 IIS로 미리 결과물을 보면서 작업할 수 때문에 상당히 편했다.

    윈도우즈 호스팅은 NHN호스팅 사이트에서 구매했다.

    결과.

    (앱은 서강대의 상징동물인 알바트로스와 상징색인 붉은색을 사용해 디자인했다)

    (전화번호와 네이버 지도 링크는 모두 html 링크 처리가 되어 출력되기 때문에 원터치로 연결이 가능하다. 사실상 AI 알로스의 핵심 기능이다)

    서강대 홈페이지에서 사용할 수 있는 서강 챗봇과 비교해서 답변의 정확성이 좋았고, 동아리나 학식 정보, 교수 평가 비교 같은 ‘비공식적’ 정보는 서강 챗봇이 잘 알려주지 못했는데 그 점에서는 알로스가 장점이 있지 않나 생각해 본다.

    25년 5월 마지막 주 수업시간에 기말 성적평가용 발표가 있었다.

    교수님께서 좋게 봐주신 덕분에 A+이 나왔다.

    그리고 대망의 경진대회 본선에서는 :

    성과 / 느낀점.

    그래도 우수상 까지는 받을 수 있을 줄 알았는데 아쉬웠다.

    알고 보니 컴공과 캡스톤 수업에서는 수업 전부터 친한 사람들끼리 팀을 짜서 나온 것 같았고 준비 기간 자체를 더 길게 가져간 느낌이었다.

    그리고 우리 팀은 팀원 4명 중 프로그래밍 기술이 있는 사람이 나까지 2명이었지만 다른 팀들은 6명이 한 팀이고 모두 전공지식을 잘 갖춘 듯 보였다.

    (심지어 인기상은 인원수 많은 컴공과 팀들끼리 서로 투표해줘서 6표가 나온 팀이 4팀… 공동 1등…)

    심사위원을 맡으신 교수님께서 ‘수업 이름에 창업이 들어가 있는데 이게 돈이 될 것 같아요?‘라고 질문하셨을 때부터 뭔가 느낌이 이상했는데 결국 장려상을 받았다. ㅋㅋㅋ

    그래도 에브리타임이 전국의 모든 대학교로 서비스를 확장한 것처럼 포켓 서강도 ‘포켓 연세’, ‘포켓 서울’ 등으로 확장할 수 있을 거라고 주장했는데 받아들여지지 않은 것 같다.

    다른 심사위원 분은 ‘쭉 둘러봤는데 다들 사용한 기술에 특별한 게 별로 없었다’는 평가를 해주셨다.

    RAG 기법은 이미 너무 흔해져서 특별할 게 없는 게 사실이지만 학생들에게 주어진 시간이나 자원이 매우 부족했다는 점을 이해해 주셨다면 좋았을 것 같다.

    그렇지만 ‘해보고 싶은 게 있었는데 그걸 구현했더니 상도 받았다!’ 라는 느낌이었기 때문에 뿌듯했고, 같이 발표 준비를 열심히 한 조원들도 ‘그래도 결과를 내서 다행이다’라는 반응이었다.

    웹 디자인 쪽은 잘 몰랐는데 아트 앤 테크놀로지 전공 조원이 디자인 초안을 잘 잡아줘서 완성품이 나왔기 때문에 ‘협력’의 중요성을 다시 한 번 느꼈다.

    여담으로 대회 당일에 발표가 끝나고 갑자기 위염이 와서 포켓 서강으로 서강대 보건실과 빠르게 통화한 후 알로스가 제공한 네이버 링크 지도를 보고 찾아가 약을 탔다.

    내게 실질적으로 도움이 되는 앱을 만들었다는 생각에 기분이 좋았다.

    마지막으로 프로그래밍 기술은 없었지만 최대한 자신들이 할 수 있는 것들을 찾아서 하려고 노력했던 조원들에게 감사한다.

    빌런이 한 명도 없는 조별 프로젝트를 이끌 수 있어서 정말 행운이었고, 나는 졸업하지만 앞으로도 모두 각자가 하고 싶은 일을 하면서 잘 살기를 소망한다.
